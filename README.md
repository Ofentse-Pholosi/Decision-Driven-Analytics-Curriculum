# Decision-Driven Analytics for MT5 EA

### Bridging Business Analysis and Data Analytics for Decision-Grade Trading Intelligence

## Overview

This project demonstrates how **Business Analysis** and **Data Analytics** can be intentionally combined to produce **decision-driven analytics**, using an MT5 automated trading system (Expert Advisor) as the applied domain.

Rather than starting with charts or dashboards, the project begins with:

* Clear decision framing
* Business-aligned KPIs
* Proper analytical data modeling

The end goal is a **decision-grade analytics dashboard**, but the dashboard is treated as the *final output* of disciplined analysisâ€”not the starting point.

This approach reflects how analytics should operate in real-world environments such as fintech, product analytics, and operational decision support.

## Why This Project

Many analytics initiatives struggle because they:

* Start with available data instead of a decision
* Produce insights without clear actions
* Build dashboards that look good but do not influence outcomes

This project was designed to address those gaps by applying **business analysis principles upfront**, ensuring that analytics directly supports an accountable decision.

The central question guiding this work is:

> *Should a standardized MT5 EA configurationâ€”restricted to Step Index synthetic instruments and governed by strict risk rulesâ€”be deployed for sustained capital allocation?*

All data analysis, modeling, and visualization exists solely to support that decision.

## Objectives

### 1. Decision Governance

* Define a clear, accountable business decision
* Explicitly document assumptions, constraints, and risks
* Avoid strategy changes driven by short-term performance noise

### 2. Business-Aligned Metrics

* Translate trading behaviour into decision-grade KPIs
* Prioritise capital preservation before profit maximisation
* Separate leading indicators (early warnings) from lagging indicators (outcomes)

### 3. Analytics Engineering Discipline

* Convert raw MT5 trade history into analytics-ready data structures
* Define correct data grain (trade, session, daily)
* Ensure metrics are auditable, reproducible, and explainable

### 4. Portfolio-Ready Analytics

* Demonstrate practical Business Analysis and Data Analytics capability
* Produce artifacts suitable for professional review
* Position the work as transferable beyond trading use cases
